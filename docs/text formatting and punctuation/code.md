!!!tldr "Rule"
    Use the inline code environment for individual commands and snippets that are less than a line of code. Otherwise, use a code block.

Like numbers and variables, code and pseudocode should not appear as plain text in a Brilliant lesson.

In Camperdown, there is a specific code display environment that is initiated by the escape \`...\`.  This inline code environment is used for single commands up to a full line of code from a program.

If a code snippet is longer than a line in a program, or if the line is long enough that it breaks across lines in the lesson, the escape \`\`\` initiates a code block.

When you include a block of code that is not pseudocode, write the name of the language in which it's written in the first line.
