# Split-lane Mode

<aside>
📏 Use `!splitlanes` when a reference image or interactive should remain on screen as the learner advances between questions.

</aside>

- The reference image or interactive is the subject of multiple questions in this block.
- Because an image or interactive cannot be removed from the left lane in `!splitlanes` mode, the current guidance is to end the block when the visual is removed from the screen. This guarantees that, by returning to the block, the learner can still access this element after they have advanced beyond it.
- **[Example](https://brilliant.org/courses/intro-neural-networks/neurons-2/binary-neurons/2/?version_id=1473)**

    ![Split-lane%20Mode%2047b2086a4a774a068ba243cf717cde43/Untitled.png](Split-lane%20Mode%2047b2086a4a774a068ba243cf717cde43/Untitled.png)


Use the left lane only if there is no single-lane solution to displaying everything learner needs to see on screen simultaneously. Minimize switching between single-lane and double-lane formats.

**[Example:](https://brilliant.org/courses/pre-algebra/factoring-2/common-factors/2/?version_id=2135)**

[Split-lane%20Mode%2047b2086a4a774a068ba243cf717cde43/Screen_Recording_2021-06-30_at_10.56.30_PM.mov](Split-lane%20Mode%2047b2086a4a774a068ba243cf717cde43/Screen_Recording_2021-06-30_at_10.56.30_PM.mov)

<aside>
✅ Learners should never lose access to images or interactives in the right lane. Begin a new block when you no longer need the image or interactive in `! splitlanes` mode.

</aside>

- Blocks usually end with a major milestone within the lesson. However, in `! splitlanes` mode, if you are ready to remove the object in the right lane, ending the block before arriving at the milestone is acceptable.
- Remember: `! splitlanes` mode is only used when an interactive or image must be visible over several questions in a sequence.

**[Example:](https://brilliant.org/courses/intro-neural-networks/neurons-2/decision-box-2/3/?version_id=1474)**
