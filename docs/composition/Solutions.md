# Solutions

<aside>
📏 If a solution requires several images, or breaks naturally into a linear set of steps, use a stepped solution. Otherwise, it's usually okay to put the solution on a single step.

</aside>

Single-step format is good for short solutions, solutions that don't require a lot of images, and solutions that feature a stepped diagrammar.

- Avoid putting stepped diagrammars in a stepped solution, since the steppers for diagrammar and solutions look similar.
- If you have more than one independent visual for a solution, then don't use a single-step solution.
- Single-step **[Example](https://brilliant.org/courses/pre-algebra/introduction-66/variables/4/)**

    ![Solutions%2060fb8316a71f47b980e96778ff09b396/Untitled.png](Solutions%2060fb8316a71f47b980e96778ff09b396/Untitled.png)


Multi-step solutions are used when your solution has multiple images or solutions that break into independent steps. Avoid putting stepped diagrammars in a stepped solution, since the steppers for diagrammar and solutions are similar.

- If a solution has an interactive diagrammar in its first step,  This is a good place for a step break to encourage engagement with the interactive.
- The second step contains the answer to the question. After the learner reads the last step of the solution, they lose their opportunity to answer the question.
- The answer should always appear on the last step of a solution so that the learner can read the earlier steps and return to answer the problem.
- The text on the second step doesn't directly refer to the first step. If a solution doesn't break cleanly into steps, use a single-step format, unless it makes the solution too long.
- Multiple-step **[Example](https://brilliant.org/courses/puzzle-science/relativity-2/bouncy-balls/2/?version_id=2029)**
