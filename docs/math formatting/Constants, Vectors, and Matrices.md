# Constants, Vectors, and Matrices

!!!tldr "Rule"
    The class of a mathematical object should be apparent from the symbol we use to represent it.

Brilliant lessons help learners develop a taste for different mathematical ideas. In our lessons,  equations are not chunks of math that break up the text. Words fall into different classes and so do mathematical objects. We'd like to reflect the class of an object in the characteristics of the symbol we choose to represent it.

Below are some of the naming conventions we aim to follow:

- Use capital letters for constants.
- Use lowercase, bold letters for vectors.
- If it's important to indicate a vector has unit length, use `\hat{}`.

The vector $\mathbf{\hat x}-\mathbf{\hat y}$ points from $(0, 0)$ to $(1, -1).$


- Use uppercase, bold letters for matrices
- If you have a vector/matrix, and you want to write its components, follow the convention in this example:

The matrix $\mathbf{A}$ is antisymmetric, so its elements obey $A_{ij} = - A_{ji}.$

- Matrices are always typeset in the centered math environment using `\begin{pmatrix} ... \end{pmatrix}`

If authoritative sources on a subject have established symbology that clashes with the guidance above, then it is more helpful to the learner if you use the more common symbol.

However, we are not trapped by prevailing standards. We should weigh the advantage of unusual choices against the wider expectation. If we are intentional about the choice we're making, there's no fundamental problem with surprising notational conventions — for example, a waves course that uses $\tau$ instead of $2\pi.$

It is essential that deviations from the prevailing standards are flagged for the learner and supported with the reasons for deviations. We are responsible for writing content that helps learners form a mapping between our conventions and the established standards. Deviations from standards should be discussed on Slack in `#editorial-lounge`.
